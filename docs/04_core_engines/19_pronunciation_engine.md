# Pronunciation Engine Design: Nomen

This document details Nomen's phonetic analysis systems, Grapheme-to-Phoneme (G2P) pipelines, and articulatory ease scoring equations.

---

## 1. Grapheme-to-Phoneme (G2P) Pipeline

Abstract startup names (e.g., "Vercel", "Retool") do not exist in standard dictionary lookups. To analyze their phonetic properties, Nomen translates letters (graphemes) to sounds (phonemes) using the open-source **`g2p-en`** python library (based on a neural network trained on the CMU Pronouncing Dictionary).

```text
              [ Input Name String: "Nomen" ]
                             │
                             ▼
                ┌─────────────────────────┐
                │   G2P Neural Network    │ (g2p_en: letters -> CMU phonemes)
                └────────────┬────────────┘
                             │
                             ▼
                [ CMU Phonemes: [N, OW1, M, AH0, N] ]
                             │
         ┌───────────────────┴───────────────────┐
         ▼                                       ▼
  ┌──────────────┐                        ┌──────────────┐
  │ IPA Mapper   │                        │ Phoneme Graph│ (Transition Difficulty)
  └──────┬───────┘                        └──────┬───────┘
         │                                       │
         ▼                                       ▼
  [ "/ˈnoʊ.mən/" ]                        [ Transition Score: 0.94 ]
```

## 2. Phonetic Transition Difficulty Algorithm

The core ease-of-speech score is determined by analyzing the transitions between sequential phoneme pairs. We train a **bigram transition matrix** on the CMU dictionary (containing 130,000+ words). 

- **Transition Score ($T_{\text{ling}}$)**:
  Let $P = [p_1, p_2, \dots, p_k]$ be the list of phonemes generated by G2P for a name.
  $$T_{\text{ling}} = \frac{1}{k-1} \sum_{i=1}^{k-1} \text{Probability}(p_{i+1} \mid p_i)$$
  - Standard consonant-to-vowel transitions (e.g., `/n/` -> `/oʊ/`) yield high probabilities.
  - Complex consonant clusters (e.g., `/s/` -> `/p/` -> `/l/` -> `/f/` as in "Splf") yield low probabilities, indicating high pronunciation difficulty.

### 2.1. Syllable Segmentation
The engine identifies syllable boundaries using vowel nuclei counts:
- Unstressed syllables: marked by vowels `/AH0/`, `/IH0/`.
- Stressed syllables: marked by `/OW1/`, `/EH1/`.
- Words with 2 to 3 syllables are optimal for memorability. 4+ syllables trigger scaling penalties.

## 4. Frontend Audio Auditioning (Web Speech API)

To bring names to life, Nomen implements a text-to-speech audition button using the native, zero-cost **Web Speech API** in browser client containers. This avoids third-party audio API costs:

```typescript
// frontend/components/branding/pronunciation-button.tsx
"use client";

import { Volume2 } from "lucide-react";

export function PronounceButton({ name }: { name: string }) {
  const handleSpeak = () => {
    if ("speechSynthesis" in window) {
      const utterance = new SpeechSynthesisUtterance(name);
      // Select natural English voice if available
      const voices = window.speechSynthesis.getVoices();
      const enVoice = voices.find(v => v.lang.startsWith("en-US")) || voices[0];
      if (enVoice) utterance.voice = enVoice;
      
      utterance.rate = 0.95; // Slightly slower for articulation clarity
      window.speechSynthesis.speak(utterance);
    }
  };

  return (
    <button onClick={handleSpeak} className="p-2 bg-blue-50 hover:bg-blue-100 rounded-full transition-colors">
      <Volume2 className="h-5 w-5 text-blue-600" />
    </button>
  );
}
```
This gives users immediate phonetic confirmation of candidate names without network overhead or third-party usage costs.
